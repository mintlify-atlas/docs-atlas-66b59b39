Broccoli is an async-first message queue library for Rust, built on Tokio. It gives you a clean, type-safe API for publishing and consuming messages across Redis, RabbitMQ, and SurrealDB — with built-in retry strategies, connection pooling, message scheduling, and fairness queues.

## Why Broccoli?

If you're familiar with Celery but want Rust's performance and compile-time type safety, Broccoli is your answer. It wraps your chosen message broker behind a single, consistent API so you can swap backends without rewriting application logic.

<CardGroup cols={3}>

## Get Started

<Steps>
  <Step title="Add Broccoli to Cargo.toml">
    ```toml Cargo.toml
    [dependencies]
    broccoli_queue = "0.4"
    tokio = { version = "1", features = ["full"] }
    serde = { version = "1", features = ["derive"] }
    ```
  </Step>
  <Step title="Build a queue">
    ```rust
    use broccoli_queue::queue::BroccoliQueue;

    let queue = BroccoliQueue::builder("redis://localhost:6379")
        .pool_connections(5)
        .failed_message_retry_strategy(Default::default())
        .build()
        .await?;
    ```
  </Step>
  <Step title="Publish a message">
    ```rust
    queue.publish("jobs", None, &my_payload, None).await?;
    ```
  </Step>
  <Step title="Process messages">
    ```rust
    queue.process_messages("jobs", Some(4), None, |msg| async move {
        println!("Got: {:?}", msg.payload);
        Ok(())
    }).await?;
    ```
  </Step>
</Steps>
